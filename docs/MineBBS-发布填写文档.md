## 插件简介

LifeLink 是一款使用 Kotlin 编写的树木生命周期托管插件，适用于 Spigot / Paper / Folia 服务端。

它的核心目标不是单纯“砍树后补一棵苗”，而是把一棵树从合法砍伐、自动补种、系统树苗保护，到再次成长为可托管树木的整个过程串成闭环，减少服主额外配置多个插件的成本。

## 核心功能

- 玩家使用配置允许的工具从树根合法砍树后，自动在原位置补种对应树苗
- 系统补种的树苗会被保护，防止被误挖、替换、液体冲掉、活塞破坏或爆炸带走
- 系统树苗成长后会自动升级为“托管树木”，下次仍可继续进入自动补种流程
- 玩家手动种植的树苗不会被强制托管，避免影响原有生存玩法
- 管理员可通过 `/lifelink saplingmode on` 把自己放置的树苗记录为系统托管树苗
- 支持甘蔗底部留苗，砍甘蔗时自动保留最底下一格
- 支持可配置的“禁止耕地被踩坏”保护开关
- 兼容 Folia 调度模型，避免跨区域不安全访问

## 支持树种

- 橡木
- 白桦木
- 云杉木
- 丛林木
- 金合欢木
- 深色橡木
- 红树木
- 樱花木
- 苍白橡木

## 指令

- `/lifelink reload`：异步重载配置与语言文件
- `/lifelink recover`：手动触发恢复检查
- `/lifelink saplingmode on|off|status`：切换管理员树苗托管模式

## 权限节点

- `lifelink.admin`：管理员总权限
- `lifelink.reload`：允许使用重载命令
- `lifelink.recover`：允许使用恢复命令
- `lifelink.admin.sapling-mode`：允许切换管理员树苗托管模式
- `lifelink.bypass.sapling-protection`：绕过系统树苗保护
- `lifelink.bypass.managed-tree`：绕过托管树砍伐限制

## 配置亮点

- 自定义允许触发自动补种的工具列表
- 潜行砍树时可跳过自动补种
- 可切换系统树苗保护与托管树限制
- 可配置补种失败后的处理策略、重试次数与延迟
- 可控制粒子效果、树木检测范围、甘蔗留苗和耕地踩踏保护

示例配置项：

- `harvest.allowed-tools`
- `harvest.sneak-bypass-replant`
- `protection.protect-system-saplings`
- `protection.protect-managed-trees`
- `protection.prevent-farmland-trample`
- `replant.failure-strategy`

## 兼容性说明

- 服务端：Spigot / Paper / Folia
- Minecraft 版本：1.20.X ~ 1.21.X
- 运行环境：Java 21

## 安装方法

1. 将插件 jar 放入服务器 `plugins` 目录
2. 启动服务器生成默认配置文件
3. 按需修改 `config.yml`
4. 使用 `/lifelink reload` 异步重载，或直接重启服务器

## 适用场景

- 生存服的树木可持续采集
- 希望减少玩家乱砍后地图秃掉的问题
- 需要保护系统补种树苗、防止被误操作破坏
- 使用 Folia，希望插件在调度模型上足够安全
